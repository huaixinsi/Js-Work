# 电影目录扩容与演示评论设计

## 目标

将当前仅同步 TMDB“正在上映”第 1 页、最多 12 部电影的逻辑，扩展为同步近两年热门电影约 60 部。首页增加可见电影数量，并为每部电影补充少量明确属于演示环境的测试评论，使课程演示页面更完整。

## 数据来源与范围

- 使用 TMDB Discover API 获取电影。
- 时间范围为执行同步当天向前两年，结束日期为执行当天。
- 按热度降序请求前 3 页，每页最多 20 部，总量上限 60 部。
- 请求使用 `zh-CN` 语言和中国区域参数。
- 过滤缺少 TMDB ID、标题、海报或有效上映日期的记录。
- 按 TMDB ID 去重，避免跨页重复。
- 保留现有 Access Token 优先、API Key 回退的认证方式。

## 后端结构

`server/domain/tmdb.js` 负责：

- 生成近两年的 Discover API 查询参数。
- 顺序读取 3 页结果。
- 合并、过滤、去重并限制为 60 部。
- 将 TMDB 数据映射为现有数据库电影字段。

`server/routes.ts` 继续负责事务写入：

- 使用 `tmdb_id` 执行插入或更新，不删除已有电影、评论或订单关联。
- 新电影自动创建一个可购票场次和对应座位。
- 同步完成后，为缺少演示评论的电影补充评论。
- 同一部电影重复同步时不重复插入同一批演示评论。

## 演示评论

- 每部电影补充 2 至 4 条测试评论。
- 评论使用现有普通演示用户 `demo`、`vipuser` 和 `plain`，不创建虚假的正式用户账号。
- 内容从多组自然中文模板中按电影 ID 稳定选择，并结合影片标题或观影感受生成差异。
- 评分保持在 7 至 10 分范围内，创建时间分散在最近若干天。
- 演示评论内容带有简短的“演示评论”标识，避免被误认为真实用户评价。
- 通过稳定内容标识或查询条件保证幂等；用户自己发表的本地评论不受影响。

## 前端展示

- 首页“热门推荐”“同城热映”“评分最高”各由 6 部增加到 12 部。
- 购票页继续展示接口返回的完整电影列表，预计约 60 部。
- 不改变卡片样式、详情页、本地评论提交和选座购票交互。
- 管理员同步完成提示显示本次获取并写入的电影数量。

## 错误处理

- 任一 TMDB 页面请求失败时，同步整体失败并返回清晰错误，不写入不完整批次。
- 缺少 TMDB 凭据时维持现有错误提示。
- 数据库电影、场次和演示评论写入保持在同一事务内，异常时统一回滚。
- TMDB 返回不足 60 部时使用所有符合条件的结果，不用虚构电影补足。

## 测试

- 测试 Discover URL 包含近两年日期、语言、地区、排序和页码。
- 测试三页结果会被合并、过滤、按 TMDB ID 去重并限制为 60 部。
- 测试 Access Token 失败时 API Key 仍可回退。
- 测试推荐分组每组最多返回 12 部。
- 测试演示评论数量为每部 2 至 4 条、内容稳定且带演示标识。
- 运行全部单元测试、TypeScript/Vite 构建和 Docker 页面响应检查。

## 验收标准

- 同步后数据库中可展示近两年热门电影约 60 部。
- 首页每个推荐分区最多展示 12 部电影。
- 每部电影详情页至少有 2 条演示评论，重复同步后数量不会持续增长。
- 原有用户评论、登录、管理员、订单、选座和支付演示功能正常。
- Docker 构建成功，`http://localhost:3000` 返回 HTTP 200。
